Web Design的趋势

Oct 30th, 2014 2:55 pm

虽然我的注意力主要在机器学习和服务端的架构，但是前端领域也在生机勃勃地发展，有很多激动人心的创意、新技术、富于洞见的设计理念和极致的美学体验。今天在网上找到一些有价值的文章，分享给大家。

Web Design: 20 Hottest Trends To Watch Out For In 2014 是一篇很棒的文章，总结了本年度（2014）年在Web Design方面可去观察的20个热点。作者自身的设计水平就很高，了解业界的动态和趋势，掌握资料翔实。如下概括这20个要点，详细的内容请参考原文。

1. Grid-Style Layouts 网格式布局

对网格设计的主流认识来自pinterest。社交网站的news feed过去常用流线型，比如twitter和Tumblr。现在象Facebook的许多页面将时间线更新散布为网格式。

这并非说要强制老的网站这么做，而是要看目的是什么。用户体验总是首先被考虑的。在使用微缩图或文字更新的场合下，网格布局能将所有内容压缩到一个易于阅读的格式，让用户可以快速略读。同时，页面内容可以保持一致并不占太多空间。

诸如开源js库masonry，可以帮你做大量繁重的工作。

没有太多的网站利用这一特性，但凡使用的，经常让视觉很舒服。

2. Crafty Image Captions 巧妙图片标题

许多网站用图片画廊来展示商品的概要、相片、微缩图等。此文讲到了独特的CSS3图片效果，其演示效果见此页面。使用图片标题(Image Caption)帮助网页访问者将更多信息与图片联系起来。而且，利用CSS3，你可以只历来浏览器自带的特性，而不需特别的javascript效果。

每个网站都应该使用自己的设计，来为读者提供便利（Each website should have its own design to provide benefit to the reader）。还有一个来自Codrps的教程关于figure numbering with CSS3。你可以自动构造标题风格的标签，自动添加到每页的图片（或图形）上。

3. Extended Form Elements 扩展表单元素

大多数前端开发者都了解jQuery的作用。它帮助你编写更平滑的javascript代码，而且更短小、更整洁。自由开源的jQuery插件有它的市场，现在的需求很旺盛。

围绕那些能增强用户体验的jQuery表单有特别的热情。这些效果包括浮动标签(floating labels)，输入校验(input validation)，指导性提示(guided tooltips)，几乎任何你可以想象到的。看一下Unheap form gallery, 它编目了开源jQuery插件，你可以下载并在新项目中测试。

4. Deeply-Focused Landing Pages 深度聚焦的Landing页面

iOS App Store刚启动时，开发者们吵闹着要发布他们下一个创意。过了几年，我们已经有几百万个针对iOS+Android设备的应用。许多这样的App甚至有它们自己的伙伴网站。

一些开发者经常购买一个域名，发布一个网站作为营销工具。这个点子后来已经扩展到包括移动游戏、开源脚本、智能手机App、事实上任何你能想到的数字产品。从根本上来说，对于鼓励潜在客户在购买前多了解产品，这些网页是必可不少的。

5. HTML5 视频播放器

虽然必要时，Flash仍有用处，但大多数Web开发者认同，Internet视频的未来是HTML5。高度推荐的两个脚本是：Video.js 和 MediaElement.js。第一个更为简单，局限于基本功能。

Media Element为你构造自己的播放器设计，提供了一些缺省的皮肤和更为优秀的文档。你也可以用此代码构造一个音乐或音频播放器。除过API之外，两个脚本都提供了文档。它们都是很好的js库。当你在未来的项目中需要自有主机的视频(self-hosted videos)时，请记着这些。

也请阅读: 10 Things Designers Need To Know About HTML5 Videos

6. 3D 渐变效果 (Transition Effects)

在过去的一年中，我已经从更多的网站中发现了一些创造性的3D动画。这些经常构建在动画效果的图片画廊、导航菜单和元素。3D效果可以用jQuery来创建，虽然CSS3正追赶过来。

不用说，所有浏览器并没有完全支持动画，设计者必须小心不要在一个页面中使用太多的动画。然而如果你愿意尝试新事物，我强烈推荐搜索这样的3D animated code examples去把玩。

7. Flat Design Elements 平面设计元素

在讨论web设计趋势时，不能忽略广泛应用的 flat UI 元素。CSS3允许设计者使用自然盒子阴影、文本阴影或圆角，创建更为平面的按钮。这一平面UI模式已经扩展到表单输入，甚至导航菜单。

但是，在不久的未来，还有更多的可以期待。很多网站已经免费发布了 平面图标集(Flat icon sets) 和 GUI kits。 在本文作者的文章 guide to flat web design，你可以找到一堆堆可免费下载的PSD/AI图像。从微软的Windows操作系统到Window Mobile Phone，地铁风格(metro-style)的布局已经普及。

也请阅读：Showcase Of Beautiful Flat UI Design

8. Personal Portraits 个人肖像

简单的个人概况网站(portfolio websites)也值得关注。你想传递一些关于你是谁，你曾在哪里，并展示你的工作样例。当为了让人们真正感兴趣，你需要建立人性的连接(human connection)。做到这点的最好途径之一是，在你页面的某个地方加入你相片的样张。

Andrea Mann 使用了一张占据主页很大幅面的相片。这也与更暗的背景融为一体，重新调色为黑白灰度(recolored to shades of B&W)。尽管你不需要在你的网站上使用大幅背景肖像(large background portraits)，但这的确很奇妙。甚至在你主页上一个小的相片或about页面，也让访问者瞥见你是谁。

9. SVG 图像或图标

基于矢量的图形(vector-based graphics)不是建立在像素，而是建立在线和坐标上。通过数学公式将单独的形状(shape)组织起来，这允许你将矢量拉伸到任何尺寸。SVG (Scalable Vector Graphics) 图像能容易被操作而不损失太多质量，在这个意义上象矢量图形。

最大的问题是所有浏览器对这些图像的支持度。许多人仍然使用IE(internet Explorer)的旧版本和其它遗留的浏览器。这是一个巨大的趋势，需要好几年掠过Internet才能到来(如果它没开始的话)。

也请阅读：More On Scalable Vector Graphics

如果你真的想开始，我可能推荐codepen, 一个开放的社区驱动的IDE，让开发者在线分享他们的工作。Snap.svg是一个JavaScript库，致力于多浏览器自然地支持SVG图像。这是有点更详细的主题，但如果你好奇，值得深入探索。

10. Unique Web Copy 唯一Web拷贝

Web开发中最酷的新特性之一是，@face-font申明。你能导入在本地或外部远端服务器的字体文件。这些字体在CSS中定义，你能在font-family属性引用它们，从而设计你的web页字体。

这已经发展到包含基于字体的图标(font-based icons)，这里灵活的字体在页面上可以象文本一样被渲染。相比真实的图形，这些图标的设计没那么良好，它可结合大量不同的颜色。但是，简单地能在页面上定制化任何字体，就让你的网站在人群中脱颖而出。

11. Lazy-Loading Animations 延迟加载动画

主题森林(themeforest)有很多WordPress主题，它有一些应用到元素上的渐变效果，但只有在你将它们滚动到视图时才会出现动画。这些元素的表现和延迟加载图片相似，除过它们已经被加载到页面但被隐藏起来。

使用JavaScript很容易探测到哪些元素在视图里，然后使用CSS3渐变或JavaScript来实现动画。

如果你想下滚动Chart.js的页面，你将发现内容+图像的块，开始慢慢地消失在视线中(fade into view)。这个趋势大概不能节省带宽，除非你有意识地等待载入内容。这更多地是关于页面美学(about page aesthetics)，为你的访问者提供柔顺的界面(a sleek interface for you visitors)。

12. Customized Image Galleries 定制的图片画廊

JavaScript和CSS不断地集成，允许开发者创造独特的新项目。Image Galleries是一个已经持续数十年的趋势。步入2014年，我们可以期待甚至对旋转木马(dynamic carousels)和图像光盒(Image lightboxes)更精彩的方案。

提供一个小例子，看一下在codrops上这个最近的画廊教程，它使用了弹性拖拽效果去在相片之间切换。这有点实验性的，但是你能看出开发者如何愿意推开边界，去看看什么是可能的。

如果你喜欢这个概念，为什么不看下jQuery画廊插件呢？这将是开始学习的更快策略，随着时间，你甚至能构建你自己的画廊插件。

13. Mega-Navigation Menus 巨大导航菜单

近年来，所有奇特的新导航种类都被测试和采用了。除过HTML5/CSS3规范，移动响应Web是这方面的大头。有一个关于mega navigation menus很小却在增长的趋势，它扩充以容纳内容和链接的大块。

这在以很大的容量发布许多独特内容的网站上，最为普遍。立即浮现脑海的，是在线杂志或web论坛。它的确占据了页面的一点空间，但它也给读者更宽广的选择去导航你的站点。

在Smashing Magazine上偶然撞见的一篇相关文章，谈到所有关于针对大型站点的导航菜单(navigation menus for mega-sites)。概念是相似的，对任何对这一趋势感兴趣的人，所提供的例子，都是好的起步点。

14. Expanding Search Bars 扩展搜索栏

回头一看，不能完全确定这一概念是否正在加速。但是，在你的布局中构造半隐藏或扩展的搜索栏(semi-hidden or expanding search bars)，正不断流行。用户点击放大镜图标，或点击进入表单本身，然后它扩展更快，允许输入更多文本。

关于这个主题，Codrops编写了很出色的教程，对任何开发者，就极具可读性。隐藏输入域首先是用于节省页面的地方。你甚至可以在顶部隐藏表单，象导航菜单一样切换它上/下(toggle it up/down like a navigation menu)。这个技术在响应式布局中最为突出，但在甚至更大、更完整的设计中也包含了这些可适应的搜索域(adaptable search fields)。

15. Featured Detail Lists 有特色的详情列表

最近，在网站上发现的featured detail lists都放在一个设计展示中。这是面向列表的详细资料，解释有关公司或产品的信息。详细资料经常伴随一些图标来以更简洁的方式说明要点。

这些详细资料列表已经成为很多主页的主要部分。有关任何产品或服务的信息的收集，给访问者一个理由去逗留。它对你提供给客户的内容给出一些洞见，它们一起如何工作(It gives some insight towards what you provide to customers and how it all works)。当快速滚动页面时，图标象糖霜(frosting)一样能吸引注意力。

16. Mobile-First Design 移动优先设计

前阵子，Luke Wroblewski的书Mobile First给出这一观点。当浏览一个站点时，不太可能轻易地区分设计过程是如果进行的。但这个想法是，首先对你的站点在移动屏幕上如何看起来如同一个响应性布局去搭建雏形(mock-up)。消减所有多余的不成功的尝试、只保留尽可能少的基本要素(Eliminate all the excess fluff and keep only the bare essentials)。

从这个观点看，将你的设计向上扩展到更为宽阔的屏幕，会非常容易。导航菜单更宽了，内容被拉长，还连同一个边栏(sidebar)。移动优先的设计将移动体验放在更高的优先级，这然后成为整个布局的基线(then becomes a baseline for the entire layout)。这个概念值得拥抱。希望设计者尝试实施它，看看它如何影响设计的工作流和最终的结果。

17. HTML5 画布

关于画布元素(canvas element)有很多可以讲的，这确实是一个研究的主题。使用JavaScript, 恰在HTML5里面就能创造出游戏或绘画应用。曾经见过，通过鼠标来收集签名的画布元素。

上面的例子来自一个简单的codepen条目。有人发布一个有光泽的加载圈到Dribble，这个笔是使用画布元素的一个仿制。另一个例子是构建在HTML5画布和JavaScript上的这个小水平加载条(horizontal loading bar)。这种可能性增长很快，我们兴奋地期待看看2~3后这样的画布会是什么。

18. Pixel Sprites & Browser Games 像素精灵和浏览器游戏

很象经典的视频播放器，浏览器游戏(browser games)过去完全在Adobe Flash中制作。但是JavaScript已经在社区中获得立足点，很多开发者很高兴去发布这类项目的开放源码。

早先，此文作者有一篇文章，详细说明了也能使用动画的像素为中心的web站点(pixel-centric websites。这些设计不常见，但能抓住你的注意力。它需要很多设计天赋来创造如此精巧的像素艺术品。因此构建你自己的浏览器内的精灵游戏，将是一个甚至更大的挑战。

刚入道时，最好的一个资源可以在HTML5 Game Engine找到。这个站点罗列了很多自由的JavaScript库用于创建你自己的浏览器游戏。关于这个过程有很多可以学习的，但幸运地是，你能在Google中找到数以百计的教程。

19. Quick User Registration 快速用户注册

有日益增加的创业公司和Web应用允许用户注册。事实上，许多服务要求你使用站点前先注册。在这个快节奏的世界，冗长的详细注册表单是极端令人扫兴的(tremendously off-putting)。

尽量让所有注册表单快捷并切中要点(Try to keep all registration forms quick and to the point)。很多更新的WebApp恰在首页包含注册表单，以抓住尽可能多的访问者。这发生在当一个访问者对你的产品好奇，看了表单只有2~4项，然后决定去注册。你可能会惊讶，这种策略对抓住新客户如此有成效。
